My Contribution & Challenging

1. Interaction system

I am one of the people who contributed to designing the basic interaction system when we needed to create it from scratch. The interaction system that I contributed included how players interact with the interactable object, zoom in on the interactable object and even place items on those interactable objects. The purpose of this system is to simplify everything and make the system easy to use and understand by other programmers.

2.inventory system and pickable

Pickable

Before I start with the inventory system, let me introduce the pickable. Pickable is a branch of interactable. Pickable was responsible for being able to be picked up by the player, used/interacted with other interactable objects, and stored their own data such as name, description, variable property, and so on. I was in charge of developing both the pickable base and the inventory system. To manage such a large scale of multiple pickable instances, a data table will be used to allow the designer to easily adjust pickable property.

Inventory system

The inventory system is a system that is responsible for storing item references, showing item information, providing item inspection and providing an interface for players to use for the interactable objects.During the inspection, the player will be allowed to zoom in and rotate or interact with the item as well. Inspection was basically using a component called SceneRender2DComponent to allow the item to be shown as an image in the widget blueprint. This will prevent items from clipping into a wall or object while the player is inspecting them.
